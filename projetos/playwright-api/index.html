// problema e decisões

Por que a estrutura foi criada desta forma

A arquitetura reduz repetição, protege a execução paralela e deixa cada falha tecnicamente explicável.

01

Contexto isolado

Cada fixture recebe seu próprio APIRequestContext. O token é compartilhado, mas o recurso descartável não.

02

Dados expressivos

Builders criam payloads válidos por padrão. Cada teste altera apenas o campo relevante para o cenário.

03

Asserts de domínio

Validações repetidas ficam centralizadas, incluindo HTTP, corpo JSON e errors[] do GraphQL.

04

Diagnóstico visível

Metadados e etapas chegam ao relatório para reduzir o tempo entre falha, análise e correção.
